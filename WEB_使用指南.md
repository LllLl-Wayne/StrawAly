# 草莓生长溯源系统 - Web前端使用指南

## 🌟 系统概述

草莓生长溯源系统Web前端是一个现代化的单页面应用程序，提供直观的用户界面来管理草莓生长记录、二维码追溯和数据统计分析。

## 🚀 快速启动

### 1. 安装依赖

```bash
# 安装Web框架依赖
pip install Flask Flask-CORS

# 或者直接安装完整依赖
pip install -r requirements.txt
```

### 2. 启动Web服务器

**Windows系统：**
```cmd
# 双击运行
start_web.bat

# 或命令行运行
python web_server.py
```

**Linux/macOS系统：**
```bash
# 添加执行权限
chmod +x start_web.sh

# 运行启动脚本
./start_web.sh

# 或直接运行
python3 web_server.py
```

### 3. 访问系统

打开浏览器访问：`http://127.0.0.1:5000`

**主要功能：**

### 📋 草莓管理

- **实时追溯**：显示草莓当前状态和历史记录
- **快速查询**：支持键盘快捷操作

**主要功能：**

- **添加记录**：为指定草莓添加观察记录
- **图片上传**：支持多种图片格式上传

**主要功能：**
- 📷 图片上传与预览
- 🌱 生长阶段选择
- 💚 健康状态记录

3. 填写观察信息

## 🛠️ 技术架构

### 前端技术栈
- **HTML5**：语义化结构
- **CSS3**：现代样式和动画
- **JavaScript (ES6+)**：原生JS实现，无框架依赖
- **响应式设计**：支持多设备访问

### 后端API
- **Flask**：轻量级Python Web框架
- **RESTful API**：标准化API接口
- **JSON数据格式**：结构化数据传输

### 主要API接口
- `GET /api/health` - 健康检查
- `GET /api/strawberries` - 获取草莓列表
- `POST /api/strawberries` - 创建新草莓
- `GET /api/strawberries/{id}` - 获取草莓详情
- `GET /api/strawberries/search` - 二维码搜索
- `POST /api/strawberries/{id}/records` - 添加记录
- `GET /api/statistics` - 获取统计数据

## ⚠️ 注意事项

### 系统要求
- **浏览器支持**：Chrome 70+、Firefox 65+、Safari 12+、Edge 79+
- **网络连接**：需要与后端服务器保持连接
- **JavaScript**：必须启用JavaScript

### 安全考虑
- **文件上传限制**：最大16MB，仅支持图片格式
- **输入验证**：前后端双重数据验证
- **错误处理**：友好的错误提示信息

### 性能优化
- **图片压缩**：建议上传前压缩图片
- **批量操作**：大量数据建议分批处理
- **缓存机制**：浏览器会缓存静态资源

## 🔧 故障排除

### 常见问题

**1. 无法访问Web界面**
- 检查Web服务器是否正常启动
- 确认端口5000未被其他程序占用
- 检查防火墙设置

**2. 数据加载失败**
- 确认数据库连接正常
- 检查后端服务器状态
- 查看浏览器控制台错误信息

**3. 图片上传失败**
- 检查文件格式是否支持
- 确认文件大小不超过16MB
- 检查网络连接状态

**4. 二维码搜索无结果**
- 确认二维码内容正确
- 检查数据库中是否存在该记录
- 尝试使用草莓ID直接查询

### 日志查看
- **服务器日志**：查看终端输出
- **浏览器日志**：打开开发者工具查看Console
- **网络请求**：使用开发者工具Network面板
